git add . | "file.name" // add files from directory in local repo
git rm . | "file.name" // remove files from dir in local repo

git pull // выгрузить все из облачного репо определенной ветки
git fetch // выгрузить все ветки
git push origin branch // залить в облако в ветку 
git commit -m "message" // делает коммит в локал репо
git remote -v // отображает все соединения с удаленными репо
git remote add origin "https://github.com/AleshaK/word_counter_module.git" // добавляет удаленный origin
git remote rm origin // удаляет все origin
rm -rf .git // удаляет гит
rm -rf .git* // удаляет гит


последовательность чтобы выгрузить файлы на гит:
1. Создаем репо на гитхабе
2. Заходим в папку с проектом и делаем:
2.1 git init -b main
2.2 git add .
3. git remote add origin "link"
4. git commit -m "comments"
5. git push origin main

smtp (работа с почтой) {
COMPOSER_MEMORY_LIMIT=-1 composer require drupal/smtp
После активации модуля надо пройти в /admin/config/system/smtp и настроить почту. 
В первом поле надо поставить on. Это активация сервиса почты. Далее идет сам почтовый сервер (провайдера зачастую, но можно использовать деф гугла).

Гугл сервис:
SMTP server - smtp.gmail.com
SMTP backup server - (можно например яндекса) smtp.yandex.ru
SMTP port - 465 (465 - защищенный, 25 - незащищенный. Gmail поддерживает только защищенные поэтому надо активировать всю защиту)
Use encrypted protocol - Use SSL
Enable TLS encryption automatically - On

в форме SMTP AUTHENTICATION необходимо указать данные аккаунта, который будет рассылать почту (в случае gmail, email гугл аккаунта и пароль).
!!!!!!!!!! в настройках гугл аккаунта, от которого ведется рассылка сообщений должна быть отключена опция:
https://myaccount.google.com/u/0/lesssecureapps?pli=1&rapt=AEjHL4MIDaLePIS1ZZTSAfhd44wDmkOmgneEDuc-NFWKZcy1rP3WFc5t0GXmxEnKRsPg_Fu7Yb9ehm4wQ_D4enWLGcbC8-r8DA


В форме email options указываются доп параметры, куда отправляются логи почты и имя, которым подписывается почта.
Остальное не нужно.
